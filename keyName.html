ZephyrJS keyNames

Zephyr adds the functionality of key fire/press tracking, through PIXI.Keys. The two functions, down(keyName) and fired(keyName) are explained in the documentation, but their keyName parameters need explanation.

The Basis for keyName Zephyr needs to store keys in an efficient way that is easily usable. For some reason, none of the default keyboardEvent properties do this. Blame Javascript.

The Solution Zephyr does a little bit of string shortening to keep the actual useful, identifying parts of keyboardEvent.code and combining similar keys (ControlLeft and ControlRight, for example)

The Code keyboardEvent.code.replace(/(Key)|(Left)|(Right)|(Digit)|(Numpad)/g, '');

Alphabet Characters
Represented by a single uppercase letter A through Z
Numeric Characters
Represented with a single numeric digit 0 through 9
Miscellaneous
Function keys are F1 through F12
Esc key is Escape
` is Backquote
